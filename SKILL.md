# 提示词增强器

## 功能描述

此技能帮助用户将简单需求陈述转化为结构化的高质量提示词，以提高大语言模型的任务完成质量。

## 适用场景

- 用户需要创建结构化的提示词时
- 用户希望优化现有提示词以获得更好结果时
- 用户需要为特定任务设计专业提示词时

## 工作流程

### 步骤1：需求分析
使用EARS（简易需求语法）对用户需求进行精确改写：
- **Environment（环境）**：在什么背景下
- **Actor（角色）**：谁需要完成
- **Response（响应）**：期望什么输出
- **Stimulus（刺激）**：触发条件是什么

### 步骤2：领域识别
识别需求所属领域，找到相关的成熟理论或最佳实践。

### 步骤3：示例提取
从需求陈述中提取具体示例或约束条件。

### 步骤4：构造增强提示词
按以下格式输出：

```
role
[根据需求填入专业角色定位]

goals/skills
[列出核心目标和技能]

workflows
[描述具体工作流程步骤]

examples
[提取的具体示例]

formats
[指定输出格式要求]
```

## 使用示例

**示例1：代码审查需求**

用户输入：
> 帮我审查这段代码有没有问题

增强后的提示词：
```
role
你是一名资深代码审查专家，精通多种编程语言和最佳实践。

goals/skills
- 识别代码中的潜在bug和安全漏洞
- 评估代码可读性和可维护性
- 提供具体的改进建议

workflows
1. 分析代码结构和逻辑
2. 检查命名规范和代码风格
3. 识别潜在问题
4. 给出改进建议

examples
无具体示例

formats
输出格式：
- 问题列表（严重程度、位置、描述）
- 改进建议
- 总体评分
```

**示例2：文档编写需求**

用户输入：
> 帮我写一个API接口文档

增强后的提示词：
```
role
你是一名技术文档工程师，擅长编写清晰、规范的API文档。

goals/skills
- 编写符合RESTful规范的API文档
- 提供完整的请求/响应示例
- 确保文档易于理解和使用

workflows
1. 分析接口功能
2. 定义请求参数和响应格式
3. 编写示例代码
4. 添加注意事项

examples
无具体示例

formats
输出格式：
- 接口名称和描述
- 请求方法和URL
- 请求参数表格
- 响应格式和示例
- 错误码说明
```

## 注意事项

1. 根据需求复杂度调整提示词详细程度
2. 确保角色定位准确
3. 工作流程要具体可执行
4. 输出格式要清晰规范
